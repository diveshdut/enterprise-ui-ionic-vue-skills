# Figma Enterprise Workflow Index

## Purpose

Use the live Enterprise Workflow Index in Figma as the primary navigation layer
for enterprise workflow design in the Ionic 8 Material UI Kit file.

Live entry point:
[Enterprise Workflow Index](https://www.figma.com/design/1wantteXXUlzPRYev1X5gr/Ionic-8-Material-UI-Kit--Community---Community-?t=hEXHxO7yI4T4m4pV-1)

This file is intentionally minimal. The live Figma index is the source of
truth for workflow categories, reusable references, pattern selection, and
design decision guidance.

Keep generic Figma technique notes out of this file. It should point to the
kit references, not restate standard Auto Layout, spacing, or master-component
usage unless the live index explicitly adds a kit-specific rule.

## How To Use

1. Start with the live Enterprise Workflow Index before exploring individual
   screens, pages, or components.
2. Use the index to choose the workflow family, closest references, and
   interaction pattern.
3. Inspect Ionic component foundations only after the workflow direction is
   clear.
4. When the user shares business screens, treat them as data reference unless
   they explicitly say they are also the UI reference.

## When Figma Is Unavailable

If the live Figma file is not accessible, do not recreate the full index here.
Instead:

- use the repo skill guidance in `SKILL.md`
- use the documented component parameter files
- use the documented workflow and pattern references available locally
- state clearly that live Figma validation was unavailable

## Maintenance

Keep this file focused on the role of the live Enterprise Workflow Index. Do
not duplicate large reference tables, node inventories, or workflow catalogs
here when that information is already maintained in Figma.
