# Figma Enterprise Workflow Reference Index

## Source File

Use the live Figma file as the visual source of truth:
[Ionic 8 Material UI Kit and Enterprise Workflow Index](https://www.figma.com/design/4IKwDgdRQTLj8QFIdhXp4K/Ionic-8-Material-UI-Kit--Community---Community-?node-id=34914-1223).

This is an open-source reference library for enterprise commerce, inventory,
logistics, administration, and planning workflows built around Ionic Material
components.

If you need to move from a design frame to implementation code, see
[design-code-mapping.md](design-code-mapping.md).

## Preservation Rule

All existing pages, frames, layers, components, styles, variables, and workflow
examples in this Figma file are reference sources. Do not edit, move, rename,
delete, restyle, or overwrite them while creating a new design unless the user
explicitly authorizes changes to identified existing content. Add new design
work in a new page or a clearly designated new area by default.

## Selection Taxonomy

| Designing A | Begin With | Reuse These Patterns |
| --- | --- | --- |
| Store-floor execution task | Fulfillment and Receiving | Task queue, scan action, progress, completion |
| Assisted selling or BOPIS order capture | Endless Aisle / BOPIS | Catalog discovery, order creation, fulfillment choice, order follow-up |
| Transfer order management | Stock Transfer, Create Shipment, View Shipment | Search/list, create transfer, shipment-style detail, discrepancies, bulk upload |
| Search or investigation workspace | Find Order and Find Shipment | Filter rail, results, sort, status chips |
| Business record detail | Order Detail and Purchase Order Detail | Summary cards, timeline, grouped items |
| Inventory adjustment or allocation | Product Inventory Detail and Pre-order Management | Variant chips, locations, quantities, correction modals |
| Setup or administrative workflow | Facility Management, Users, Settings | Configuration cards, permissions, related records |
| Rule builder or routing policy | Brokering Rules Builder | Multi-column rule construction, sequencing, actions |
| Recurring processing controls | Job Manager | Schedule cards, state toggles, job details |
| Bulk upload or reconciliation | Import / Export Management | Mapping, validation, remediation, bulk action |

## Manufacturing Execution And Planning Mapping

Use these references when designing production-run workflows:

| Manufacturing View | Begin With | Why |
| --- | --- | --- |
| Create Production Run | Create Shipment and Create Facility | Structured inputs, material/product choice, quantity entry, final create action |
| Find Production Runs | Find Order and Find Shipment | Date/status/facility filtering, result cards, sorting |
| View Production Run | Purchase Order Detail and Product Inventory Detail | Summary, material quantities, inventory visibility, timeline |
| Execute Production Operations | Receiving and Cycle Count | Scanning, confirmed quantities, progress, exceptions |
| Plan Rules or Scheduled Runs | Brokering Rules Builder and Job Manager | Routing configuration and recurring processing controls |

## Shared Pattern Vocabulary

- `Filter + Results`: use for finding or investigating records.
- `Master + Detail`: use for selecting an entity and focusing an editor or
  detailed inspector.
- `Execution Task`: use for associates performing scans, quantity work, and
  completion steps.
- `Configuration Cards`: use for settings and setup screens with scoped actions.

## Maintaining The Index

Add or extend the Figma index when a designed workflow introduces a reusable
interaction pattern that is not adequately represented by existing frames.
Keep this reference document aligned with any durable additions to the Figma
index.
